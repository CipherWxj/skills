# 步骤3: 综合评分与报告生成

## 目标

将结构问题和内容问题去重汇总，计算最终得分，输出面向用户可读的 PRD 质量检测报告。

## 输入

- `structure-review.json`
- `content-review.json`
- `assets/03-review-template.md`

## 分工边界

本 prompt 只负责综合评分、跨阶段去重和最终报告组织；不重新定义结构或内容规则。

- 结构判断标准来自 `assets/01-doc-structure.yaml` 与 `structure-review.json`。
- 内容判断标准来自 `assets/02-content-quality.yaml` 与 `content-review.json`。
- 报告章节和表格骨架来自 `assets/03-review-template.md`。
- 本阶段只对 findings 做二次治理：去重、过滤、排序、扣分、归纳待确认点和可测性结论。

## 执行步骤

### 1. 规范化 finding

- 读取 `structure-review.json`、`content-review.json`。
- 每条 finding 统一成：`dimension`、`rule_id`、`severity`、`problem`、`impact`、`confirmation_question`、`evidence`。
- `evidence` 只保留人类可读原文；若阶段产物仍带 `<cite>`、`<img>`、HTML/Markdown 标签、用户 mention 包装或其他展开占位，生成报告前先去除这些标签，只保留可读文本。
- `problem` 写具体缺口；`impact` 写影响的评审、研发或测试动作；`confirmation_question` 写 PRD 作者需要回答的问题。

### 2. 过滤与去重

- 过滤模板占位、协作空表、附录 TODO、已被评论明确覆盖的问题、删除线内的废弃内容问题以及无权限文档/不可读外链导致的问题；过滤项不扣分、不计入命中统计，只写入“非阻断说明”。
- 按根因去重。同一证据、同一缺口、同一待确认点只保留一条；结构和内容重复时，保留更能解释测试风险的一条。
- 合并后严重度取最高值；扣分只扣一次。
- 报告表格只展示一个最贴近主风险的规则编号，禁止用逗号拼多个规则编号。

### 3. 计分

扣分规则：

| 维度 | high | medium | low |
| ---- | ---- | ------ | --- |
| 结构 | 5    | 3      | 1   |
| 内容 | 6    | 4      | 2   |

计算方式：

1. 仅使用过滤和去重后的 findings 扣分。
2. 结构得分 = `max(40 - 结构扣分合计, 0)`。
3. 内容得分 = `max(60 - 内容扣分合计, 0)`。
4. 总分 = 结构得分 + 内容得分。
5. 最终等级只按总分区间判断：
   - `85-100`：优秀
   - `70-84`：合格
   - `60-69`：待改进
   - `0-59`：不建议进入评审

不要用 high 问题、可测性结论或主观判断改写最终等级；不要输出主观判断，只输出综合评分、质量等级、结构得分、内容得分、可测性结论。

### 4. 生成报告

按 `assets/03-review-template.md` 输出 `final-report.md`：

- 最终报告必须覆盖：文档来源与检测时间、综合评分与等级、结构完整性问题表、内容质量问题表、待确认问题、可测性评估、非阻断说明、规则命中统计、历史知识库使用情况。
- 总览：综合评分、质量等级、结构得分、内容得分、可测性结论。
- 维度得分：分数、扣分、命中数、high/medium/low 统计。
- 问题表：按严重度 high → medium → low 排序；同级内按结构优先、测试阻断程度、原文顺序排序。
- 待确认问题：提炼 3-5 条 Top 决策问题，不逐条复制问题表。
- 可测性评估：沿用 `content-review.json.testability`，结合 `main_gaps` 和关键 finding 总结。
- 非阻断说明：列出被过滤但值得提示的占位、空表或评论覆盖项。
- 历史知识库使用情况：说明是否使用历史知识库；如使用，列出命中的文件和影响；如未使用，明确写“未使用历史知识库”。
- 附录统计：结构命中、内容命中、过滤数量、合并数量。

输出要求：

- 只落盘 `final-report.md`，不要落盘 `final-score.json`。
- 总分、等级、维度得分和命中统计必须能从去重后的 findings 反推。
- 若某维度无问题，写“未检出问题”。
